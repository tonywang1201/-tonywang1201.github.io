使用Group Manager插件控制服务器权限

1. 下载插件

去spigot论坛下载插件

2. 安装插件

把GroupManager.jar放入plugins文件夹

重启服务器即可

3. 配置插件

配置文件

请参见:Group Manager/配置 第一次下载Group Manager时，一般只有.jar文件。在第一次运行Group Manager时，它会生成可以编辑的配置文件。 标准的配置和范例里详细的布局很相似，但在使用之前至少需要更改一点，需要将自己加入管理员。
第一次生成的配置文件时的文件目录结构会看起来像这样:

常用命令

  • /manuadd: 移动⽤户到希望的⽤户组。（如果不存在的话会添加⽤户组到⽂件）

    • /<命令> <玩家> <⽤户组>
    • 权限名：groupmanager.manuadd

  • /manwhois: 获得⽤户处在的⽤户组。

    • /<命令> <玩家>
    • 权限名：groupmanager.manwhois

  • /manuaddp: 将权限直接添加到玩家。

    • /<命令> <玩家> <权限>
    • 权限名：groupmanager.manuaddp

  • /manudelp: 从玩家直接移除权限。

    • /<命令> <玩家> <权限>
    • 权限名：groupmanager.manudelp

  • /manulistp: 列出⼀个玩家的所有权限。

    • /<命令> <玩家>
    • 权限名：groupmanager.manulistp

  • /mansave: 在⽂件上保存所有权限的更改。

    • /<命令>
    • 权限名：groupmanager.mansave

注意在这⾥使⽤ g: 前缀的格式修改全体⽤户组是有效的。

  • /mangadd: 将⼀个⽤户组加⼊到系统。
    • /<命令> <⽤户组>
    • /<命令> <g:全体用户组>
    • 权限名：groupmanager.mangadd

其余命令可以查看中文wiki以及mcbbs，csdn (上方指令也取自其中)

4. 实际使用场景

插件启动后，查看默认配置文件，默认的权限组是这样的

在玩家Alanxu加入游戏后，他被默认分配到了Default权限组，在users.yml里可以看到他处于的权限组和权限

默认Default组的权限是不够的，所以这里用指令手动添加一些常见的权限。

manuaddp Alanxu essentials.home

manuaddp Alanxu essentials.sethome

manuaddp Alanxu essentials.tp

manuaddp Alanxu essentials.tp.others

赋予玩家权限

这时users.yml 里可以看到他已经拥有了新加的权限

  • 值得注意的是，用户所在的权限组也决定了他是否有足够的权限使用Group Manager指令来赋予他人或自己权限

spigot服务器安装EssentialsX插件

EssentialsX插件资源汇总：https://www.mcbbs.net/thread-1369748-1-1.html

EssentialsX 是一个基于 Spigot 服务端的基础插件，为从大到小的服务器提供核心功能。这些功能包括： 玩家可以自由设置家 服务器传送或给玩家提供物品套组，可以设置跨世界或单独世界。 玩家与玩家间的私有消息，传送，发送传送请求 玩家自定义昵称 很多的管理员工具包括踢出服务器，临时禁止登陆服务器、禁言与监禁 内建经济系统，包括木牌商店、付费执行命令和完全的 Vault 支持

此外，EssentialsX 的选择模块提供了更多综合的功能如聊天、世界保护、GeoIP查找还有更多

EssentialsX 是原先 Essentials 项目 的分支，Essentials 已于2014年停止更新。EssentialsX 相比 Essentials 和 Spigot 上的 Essentials 分支添加了一些新特性。

1.在EssentialsX官网下载插件

EssentialsX基础系列插件包括多个.jar文件：

  • EssentialsX
  • EssentialsX Chat
  • EssentialsX Spawn
  • EssentialsX Discord
  • EssentialsX AntiBuild
  • ……

下载后把文件放入服务器目录下的plugins文件夹

2. 启动插件

启动插件后，文件夹里会出现Essentials文件夹，里面包含了控制功能的.yml文件

3. 使用插件

直接进游戏连接服务器，就可以使用插件提供的额外命令。由于插件提供的功能很多，后续可以用其他插件来控制权限。

有名的权限控制插件有很多，例如：

GroupManager插件：
给权限组添加权限：/mangaddp 权限组(默认default) 权限
删除权限组权限：/mangdelp 权限组(默认default) 权限 或 /mangaddp 权限组(默认为default) -权限(注意权限前面的减号)
给单独玩家添加权限：/manuaddp 玩家ID 权限
删除单独玩家权限：/manudelp 玩家ID 权限 或 /manuaddp 权玩家ID -权限(注意权限前面的减号)

LuckPerms插件：
给权限组添加权限：/lp group 权限组(默认default) permission set 权限 true
删除权限组权限：/lp group 权限组(默认default) permission set 权限 false
给单独玩家添加权限：/lp user 玩家ID permission set 权限 true
删除单独玩家权限：/lp user 玩家ID permission set 权限 false

(参考资料：https://www.bilibili.com/read/cv9455406)

其中，GroupManager是Essentials插件中的一个分支，是许多人常用的一个权限组管理。现在似乎已是一个独立的插件了？

我的世界服务器自动备份存档

家庭服务器里用centos搭建了我的世界java版服务器，玩的多了就准备1增加自动备份的功能。由于用的centos7，准备使用crontab服务定时备份。上网搜索了一遍发现shell脚本有以下几种：

  • 用tmux维持服务器后台运行

  • 用命名管道在服务端后台运行的同时向其发送指令

  • 用screen保持后台运行并向其发送命令

  • 直接用shell脚本复制并压缩存档文件

后来找到一篇文章说“有很多教程直接对 world 文件夹进行归档，然而这是存在一定的风险的，因为你在归档的同时服务器仍在运行，归档有可能是残缺的。“加上自己本来开服就用到了screen工具，最后决定用关于screen的脚本。

代码参考了bilibili作者：银靴 https://www.bilibili.com/read/cv11778380?ivk_sa=1024320u

专栏中还提到可以用rsync恢复存档

通过rsync可以快速恢复存档，用tar解压备份存档后，rsync可以快速检测备份与游戏文件之间的差异，并迅速替换不同的文件，相比copy paste更快捷方便

但脚本运行完后后机器上出现了两个screen

检查了一遍怀疑是screen命令有一些问题。一开始新建screen用的是screen -S name，但后来发现这样新建的窗口在输入stop命令后不会自动关闭，在接下来重启任务后就会出现两个同名的窗口。而使用screen -dmS 名字 命令新建窗口时，窗口在输入stop后会自动关闭，就不会有这个问题。（关于screen -dms）
